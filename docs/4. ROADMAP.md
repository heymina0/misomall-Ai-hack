## Phase 1 — 검색 기본 구조

> 목표: 키워드 입력 시 의약품 상품 목록 노출 + 우선순위 정렬 기본 구조 완성

### Sprint 1: 검색 결과 레이아웃 + 정렬

| 태스크 | 설명 | 검증 방법 |
|--------|------|-----------|
| 검색창 고정 영역 | 상단 검색창 유지, 입력 키워드 표시 | 스크롤 시 검색창 유지 확인 |
| 상품 카드 컴포넌트 | 상품명, 제조사, 규격, 가격, 할인율 노출 | 카드 UI 렌더링 확인 |
| 정렬 옵션 | 판매순 / 신상품순 / 낮은가격순 | 정렬 선택 시 목록 순서 변경 확인 |
| 검색 결과 우선순위 | 1~4순위 정렬 로직 적용 | 완전일치 상품 상단 노출 확인 |

**완료 기준**
- ✅ 검색어 입력 시 상품 목록 노출
- ✅ 우선순위 1~4순위 정렬 동작
- ✅ 정렬 옵션 동작

---

## Phase 2 — 의약품 정보 기반 검색

> 목표: 원스글로벌 API 기반 효능·성분 매칭 검색 + 자동완성 + 연관검색어

### Sprint 2: 원스글로벌 API 연동 + 의약품 정보 검색

| 태스크 | 설명 | 검증 방법 |
|--------|------|-----------|
| 원스글로벌 API 연동 | 성분명·효능·효과 데이터 검색 매칭 | `해열`, `근육주사` 검색 시 관련 상품 노출 확인 |
| 검색 범위 확장 | 상품 데이터 + 의약품 정보 통합 검색 | 성분명 단독 검색 결과 확인 |
| 자동완성 | 상품명 우선 + 성분·효능 기반 확장 | 입력 시 자동완성 0.5초 이내 응답 확인 |

### Sprint 3: 연관검색어

| 태스크 | 설명 | 검증 방법 |
|--------|------|-----------|
| 연관검색어 영역 | 최소 1개 ~ 최대 7개 칩 노출 | 노출 개수 확인 |
| AX 키워드 우선 노출 | 진료과 필터 적용, 최대 3개 | AX 3개일 때 관리자 키워드 4개 노출 확인 |
| 클릭 동작 | 클릭 시 해당 키워드로 재검색 | 검색창 키워드 변경 확인 |

**완료 기준**
- ✅ 효능·성분 기반 검색 결과 노출
- ✅ 자동완성 0.5초 이내 응답
- ✅ 연관검색어 우선순위 로직 동작

## Phase 3 — 개인화 + 전환율

> 목표: 재구매 편의성 + 이벤트 노출 + 대체의약품 모달

### Sprint 4: 구매이력 + 이벤트 배너

| 태스크 | 설명 | 검증 방법 |
|--------|------|-----------|
| 내 구매 상품 영역 | 최대 3개 + 초과 시 슬라이더 | 4개 이상 이력 시 슬라이더 동작 확인 |
| 노출 조건 필터 | 현재 검색 결과 포함 + 구매 이력 교집합 | 검색 결과 없는 구매 상품 미노출 확인 |
| 이벤트 배너 | 상품 3개↑ + 관련 이벤트 시 노출 | 상품 수에 따른 배너 위치 확인 |
| 페이지네이션 | 페이지당 최대 80 슬롯 | 이전/다음/처음/마지막 동작 확인 |

### Sprint 5: 대체의약품 모달

| 태스크 | 설명 | 검증 방법 |
|--------|------|-----------|
| 모달 호출 | 검색 결과 / 상품 상세에서 버튼 클릭 시 노출 | 두 진입점 모두 동작 확인 |
| 원스글로벌 API 조회 | 동일 성분·유사 약효 상품 조회 | 대체 의약품 리스트 노출 확인 |
| 구매한 상품만 보기 토글 | ON 시 구매 이력 상품만 필터링 | 토글 동작 + 결과 없을 시 안내 메시지 확인 |
| 상품 상세 이동 | 선택 시 상품 상세 페이지 이동 | 페이지 이동 확인 |

**완료 기준**
- ✅ 구매이력 상품 상단 노출
- ✅ 이벤트 배너 조건별 노출
- ✅ 대체의약품 모달 동작
- ✅ 토글 필터링 동작
