## Sprint 1: 전체 기능 구현 (2026-03-13)

> 기간: 3일 (2026-03-13 ~ 2026-03-15)
> 목표: 검색 기본 구조 + 의약품 정보 기반 검색 + 개인화 기능 전체 설계 완료

### 태스크

| 태스크 | 설명 | 검증 방법 | 상태 |
|--------|------|-----------|------|
| 검색창 고정 영역 | 상단 검색창 유지, 입력 키워드 표시 | 스크롤 시 검색창 유지 확인 | ⬜ Pending |
| 상품 카드 컴포넌트 | 상품명, 제조사, 규격, 가격, 할인율 노출 | 카드 UI 렌더링 확인 | ⬜ Pending |
| 검색 결과 우선순위 | 1~4순위 정렬 로직 적용 | 완전일치 상품 상단 노출 확인 | ⬜ Pending |
| 원스글로벌 API 연동 | 성분명·효능·효과 데이터 검색 매칭 | `해열`, `근육주사` 검색 시 관련 상품 노출 확인 | ⬜ Pending |
| 자동완성 | 상품명 우선 + 성분·효능 기반 확장 | 입력 시 자동완성 0.5초 이내 응답 확인 | ⬜ Pending |
| 연관검색어 | 최소 1개 ~ 최대 7개 칩 노출, AX 키워드 우선 | AX 3개일 때 관리자 키워드 4개 노출 확인 | ⬜ Pending |
| 구매이력 상품 영역 | 최대 3개 + 초과 시 슬라이더 | 4개 이상 이력 시 슬라이더 동작 확인 | ⬜ Pending |
| 이벤트 배너 | 상품 3개↑ + 관련 이벤트 시 노출 | 상품 수에 따른 배너 위치 확인 | ⬜ Pending |
| 대체의약품 모달 | 동일 성분·유사 약효 상품 조회 + 토글 필터 | 두 진입점 동작 + 토글 필터링 확인 | ⬜ Pending |

- ✅ 효능·성분 기반 검색 결과 노출
- ✅ 자동완성 0.5초 이내 응답
- ✅ 연관검색어 우선순위 로직 동작
- ✅ 구매이력 상품 상단 노출
- ✅ 대체의약품 모달 동작
