# AI_CONTEXT.md
> AI 기능 설계 및 구현 시 일관된 컨텍스트를 유지하기 위한 프로젝트 정의 문서.

---

## 1. 해결하는 문제

의료기관 종합몰(미소몰)의 의약품 검색은 상품명 중심으로만 동작.

그러나 실제 사용자는 상품명을 알고 검색하는 경우 보다 **치료 목적 기반**으로 의약품을 탐색하는 경우가 많음.

| 상황 | 예시 |
|------|------|
| 특정 성분 의약품이 품절된 경우 | `아세트아미노펜` 계열 대체 의약품 탐색 |
| 동일 효능 의약품을 찾는 경우 | `해열`, `진통` 기준 탐색 |
| 투여 방식 기준으로 찾는 경우 | `근육주사` 투여 가능 의약품 탐색 |

현재 검색은 이 탐색 흐름을 지원하지 못함.

---

## 2. 핵심 개선 방향

상품명 매칭 중심 → **의약품 정보(효능·성분·투여 방식) 기반 대체 탐색** 지원

---

## 3. 사용자

| 구분 | 사용자 | 검색 목적 |
|------|--------|-----------|
| 주요 | 의사 | 치료,투여 목적 기반 의약품 탐색 |
| 주요 | 간호사 | 투여 방식 기반 의약품 탐색 |
| 주요 | 의료기관 주문 담당자 | 대체 의약품 주문 |
| 관리 | 관리자 | 연관검색어 관리 |

---

## 4. 검색 데이터 구조

```
검색 입력
    ├── 상품 데이터
    │     상품명 / 카테고리(소분류) / 규격 / 제조사 / 공급사 / 보험코드
    │
    └── 의약품 정보 데이터 (원스글로벌 커넥트DI API)
          성분명 / 효능·효과 (pharmaceutical_effects_ko)
```

---

## 5. 검색 로직

```
1순위  상품명 완전 일치 (공백 무시)
2순위  상품명 부분 포함
3순위  의약품 정보 매칭 (효능·성분)
4순위  상품 부가정보 매칭 (제조사 > 규격 > 단위 > 특이사항 > 공급사)
```

---

## 6. 검색 결과 화면 구성

```
[내가 구매했던 상품]   ← 구매 이력 있는 경우만 노출 (최대 3개, 초과 시 슬라이더)
────────────────────
[연관 검색어]          ← 최대 7개 (AX 키워드 우선, 관리자 키워드 보완)
────────────────────
[일반 검색 결과]       ← 우선순위 정렬 적용
    + [이벤트 배너]    ← 상품 3개 이상 + 관련 이벤트 존재 시 노출
────────────────────
[페이지네이션]         ← 페이지당 최대 80 슬롯
```

---

## 7. 구현 제약 및 범위

**포함 범위**
- 의약품 정보 기반 검색 로직
- 검색 결과 정렬 기준 정의
- 검색 결과 UI 구조 설계

**제외 범위**
- 원스글로벌 API 신규 개발 (기존 연동 활용)
- 관리자 기능 개발
- 모바일 환경 지원

**기술 제약**
- 검색 응답속도 2초 이내
- 자동완성 응답속도 0.5초 이내
- 지원 환경: PC Web
